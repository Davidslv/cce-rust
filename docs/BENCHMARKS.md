## Interpretation

With the hashing embedder, retrieval is essentially lexical (SPEC §10 note), so recall reflects keyword overlap between the query and code identifiers rather than semantic similarity. Token savings are large because a query returns a handful of function-sized chunks instead of the whole files they live in, which is exactly the point of the engine: feed a model precise snippets, not entire files. Recall and token-savings numbers are deterministic and should match the Ruby implementation on the same corpus; latency is language-specific.
