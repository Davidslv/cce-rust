# Benchmarks (v2 — language packs)

Generated by `cce bench --lang <language>` using the default deterministic
**hash** embedder. The four languages in active use (SPEC-V2 §8) are benchmarked
against a pinned real repository each; Python and JavaScript remain validated
packs but ship no labeled benchmark corpus.

## Methodology (whole-repo indexing)

`cce bench` indexes the **whole repository exactly as `cce index` does**
(SPEC-V2 §8): the normal walker (base SPEC §7.1 ignore rules — `.git/`, `.cce/`,
`node_modules/`, `.venv`/`venv/`, `__pycache__/`, `dist/`, `build/`, any dotdir;
skip files > 2 MB and non-UTF-8), then pack-matched files are AST-chunked and
**every other text file becomes a fallback `module` chunk**. There is no
extension pre-filter — the `--lang` argument selects only the labeled query set
and this report's language label, not the file set. Because both implementations
index the identical file set with identical chunking, **Recall@5/@10 and token
savings are byte-for-byte identical across cce-ruby and cce-rust** on the same
pinned corpus; only index/query latency differs by engine.

## Interpretation

With the hashing embedder, retrieval is essentially lexical (SPEC §10 note), so
recall reflects keyword overlap between the query and code identifiers rather than
semantic similarity. Indexing the **whole** repository (not just the target
language's sources) is what makes the two implementations comparable — the file
set is defined identically — but it also lowers the headline numbers versus a
language-only index: the corpus now includes docs, configs, tests, and
other-language files, so a query has more chances to surface a near-miss (recall)
and result files that are chunked as whole-file `module` fallbacks contribute no
savings for that file (token savings). Ruby and C still save the most (72–77%)
because their target files are large relative to the functions/structs the chunker
isolates; Rust and TypeScript save less (42–48%) because hyperfine's and zustand's
files are already small and function-dense, and many small config/text files enter
the baseline at full size.

Recall and token-savings depend only on the corpus and the chunking, so the Ruby
implementation gets **identical** figures on the same pinned corpora (a
cross-language cross-check); index throughput and query latency are engine-specific
(Rust's are shown here).

> Reproduce a single language with, e.g.,
> `cce bench /path/to/sinatra --lang ruby --name "sinatra/sinatra@v4.1.1"`.
> The command indexes the whole repo and writes this file (one language per
> invocation); the table above collates the four runs.
